Day 6 八股文详细答案（Java 基础进阶）

1. 反射的基本原理是什么？如何通过反射创建对象和调用方法？

反射是指在运行时动态获取类的信息（属性、方法、构造器等）并操作它。

2. 反射有哪些典型应用场景？

  • Spring 依赖注入（IOC）
  • ORM 框架（MyBatis 反射注入属性）
  • JUnit 测试框架（动态调用测试方法）
  • 工厂模式实现（通过类名实例化对象）

3. Java 中的泛型是如何实现的？什么是类型擦除？

Java 泛型是 伪泛型，编译期存在，运行期被擦除（Type Erasure），即：

类型擦除会将泛型替换为其 上限（默认是 Object），并插入强制类型转换。

4. 泛型中的 <? extends T> 和 <? super T> 有什么区别？

通配符 意义 适用场景
? extends T 上限通配符，T 或子类 只读，不能写
? super T 下限通配符，T 或父类 可写入 T 或子类，读取为 Object

5. 自动装箱和拆箱的原理是什么？可能带来哪些问题？

  • 自动装箱：基本类型 → 包装类型，如 int -> Integer
  • 自动拆箱：包装类型 → 基本类型，如 Integer -> int

6. 包装类型之间的 == 比较和 .equals() 有什么区别？

  • == 比较的是引用地址，除非是 [-128, 127] 之间的缓存 Integer。
  • .equals() 比较的是值是否相等。

7. 什么是常量池？它包含哪些内容？

  • 常量池是 JVM 用来存放编译期间确定的各种常量（字符串字面量、final 常量等）。

  • 包含：

    • 字符串常量池（String Pool）
    • 类、字段、方法的符号引用
    • 编译期常量表达式（如 final int x = 5 + 3）

8. String 为什么是不可变的？这样设计的好处是什么？

  • 字符串被声明为 final，内容存在字符数组中且不能修改。

  • 好处：

    • 安全（如用作网络参数、数据库 key）
    • 可缓存（常量池机制）
    • 线程安全
    • 实现 hashCode 缓存，提高性能

9. new String("abc") 和 "abc" 在内存中的区别？

  • "abc"：放入字符串常量池中。
  • new String("abc")：在堆中创建一个新对象，常量池中的字符串作为构造参数。

10. 枚举类的本质是什么？它有哪些常用功能？

  • 枚举是继承 java.lang.Enum 的特殊类，本质是一个 final 的类，包含一组固定实例。

  • 特点：

    • 线程安全，单例
    • 可添加构造方法、字段、方法
    • 可用于 switch 语句
    • 内置方法：name()、ordinal()、values()、valueOf(String name)
