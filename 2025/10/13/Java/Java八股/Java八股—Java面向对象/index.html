Java八股—Java面向对象

2025.10.13

面向对象程序设计概述

  • 面向过程：先确定如何操作数据，再决定数据的结构，适用于小规模问题。
  • 面向对象OOP：先确定数据的结构，再决定操作数据的算法，适用于大规模问题。

类

  1. 类是构造对象的模板或蓝图

  2. 封装是处理对象的一个重要概念：就是将数据和行为组合在一个包中，并对对象的使用者隐藏具体的实现方式

  • 实例字段：对象中的数据
  • 方法：操作数据的过程
  • 对象的状态：特定对象有一组特定的实例字段值，这些值的集合就是这个对象的当前状态

  1. OOP原则：

    • 封装：决不能让类中的方法直接访问其他类的实例字段
    • 扩展：可以通过扩展其他类来构建新类

  2. 继承

    • 继承：基于已有的类创建新的类。继承⼀个类就是复⽤（继承）这些类的⽅法，⽽且可以增加⼀些新的⽅法和字段
    • 继承：is-a 关系

  3. 类、父类和子类

    1. extends 关键字
    2. 父类：又叫基类、超类
    3. 子类：又叫派生类、孩子类
    4. 子类比父类拥有更多的功能
      1. 通过扩展父类定义子类时，只用指出子类与父类的不同之处
      2. 一般的方法放在父类中，特殊的方法放在子类中

  4. 覆盖方法

    • 超类中的有些方法对子类并不一定适用，此时需要在子类中覆盖（重写）父类方法
    • 使用super关键字可以调用父类的方法，避免覆盖父类方法时调用父类方法造成不必要的递归
    • super不是一个对象的引用，不能将super赋值给一个对象变量

  5. 子类构造器

    1. super(…)：调用父类的构造器
    2. 必须放在子类构造器的第一行
    3. 如果没有显式调用super(…)，Java会自动调用父类的无参构造器
    4. 如果父类没有无参构造器，必须在子类构造器中明确指明调用父类的哪个构造器，否则编译器会报错

  6. this与super关键字

    • this关键字
      • 隐式参数的引用
      • 调用该类的其他构造器
    • super关键字
      • 调用父类的方法
      • 调用父类的构造器
    • this可以作为当前对象的引用，但是super不能作为父类对象的引用

对象

  1. 对象的状态改变必须通过调用方法实现（如果不经过方法改变对象状态，说明破坏了封装性）
  2. 对象的状态不能完全描述一个对象，每个对象有一个唯一标识。（在面向对象编程中，对象的状态通常由其属性值来体现，但这些状态并不能完全描述一个对象。除了状态之外，每个对象还拥有一个唯一标识，这个标识用于区分不同的对象实例。即使两个对象的状态完全相同，它们依然是不同的个体，因为它们在内存中的地址或引用不同。唯一标识确保了对象的独立性和可追踪性，是对象区别于其他对象的重要特征）
  3. 作为同一个类的实例，每个对象的标识总是不同的，状态往往也存在着差异

类之间的关系

  1. 继承关系：is-a 关系：一个更特殊的类是一个更一般类之间的关系
  2. 依赖关系：uses-a 关系：一个类的方法使用另一个类的对象
  3. 聚合关系：has-a 关系：一个类包含另一个类的对象作为其属性

继承与多态

1. 继承层次

  • 由一个公共超类派生出的所有类的集合称为继承层次
  • 在继承层次中，从某个特定的类到其祖先的路径称为该类的继承链（子到父）
  • Java不支持多继承，但是可以通过接口来实现

2. Java中，对象变量是多态的

  • 父类类型的变量既可以引用自身类型的变量，也可以引用子类类型的变量
  • 子类类型的变量只能引用自身类型的变量，不能引用父类类型的变量

3. 子类可以调用父类的public、protected、包权限的方法，但是父类不可以调用子类的特有方法

  • 纯父类变量（左右都是父类）不可以调用子类特有方法
  • 上转性变量（左边是父类，右边是子类）可以调用子类重写父类的方法，但是仍然不能调用子类独有的方法

    重载解析：在所有名为f的⽅法中，找到⼀个与所提供参数类型完全匹配的⽅法
    此时，编译器已经知道需要调⽤的⽅法名字和参数类型
  3. 静态绑定（编译阶段绑定）
    如果是private⽅法、static⽅法、final⽅法或者构造器，那么编译器将可以准确地知道应该调⽤哪个⽅法
    此时调⽤⽅法只⽤考虑x的类型（若在类C中找不到f⽅法，则向其⽗类中找），不需要考虑类C的⼦类（因为这⼏种修饰的⽅法都不能被继承）
    只有静态绑定成功，即编译通过了，才能进⼊运⾏阶段

5. 阻止继承：final类和方法

  1. final修饰字段
    • 基本类型：不可更改
    • 引用类型：不可指向新的引用，但是对象状态可能会改变
  2. final修饰类：该类不能被继承
  3. final修饰方法：该方法不能被覆盖
    将方法或类声明为final的主要原因：确保它们不会在子类中改变语义

6. 强制类型转换

  1. 向上转型（upcasting）
    • 子类型->父类型
    • 又被称为：自动类型转换
  2. 向下转型（downcasting）
    • 父类型->子类型
    • 又被称为：强制类型转换
  3. 进行强制类型转换的唯一原因：在暂时忽视对象的实际类型之后，使⽤对象的全部功能。⽆论是向上转型还是向下转型，这两种类型之间必须要有继承关系；否则，编译器报错。
  4. ⼩结：
    1. 只能在继承层次内进⾏类型转换
    2. 在将超类转换成⼦类之前，应该使⽤instanceof进⾏检查。
  5. 通过类型转换调整对象的类型并不是⼀种好的做法。在⼀般情况下，应该尽量少⽤强制类型转换和instanceof运算符
  6. ⼤多情况下，因为多态性的动态绑定机制，我们不⽤将Employee转换为Manager也能正确调⽤⽅法；只有在使⽤Manager特有⽅法才需要进⾏强制类型转换。例如，setBonus⽅法。但是，此时应该⾃问超类设计是否合理。是否需要重新设计超类，并添加setBonus⽅法，这才是更合适的选择。
