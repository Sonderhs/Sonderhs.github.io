Day05 八股文详细答案（Java 基础：面向对象 & 异常机制）

1. Java 中的继承和接口有什么区别？

特性 继承（extends） 接口（implements）
关键词 extends implements
数量限制 只能继承一个类 可以实现多个接口
设计目的 代码复用 规范行为，实现多态
包含内容 可有构造器、普通方法、字段等 只能有常量、抽象方法（Java 8+支持默认方法）

2. 抽象类和接口的区别？

特性 抽象类 接口
是否可以有构造方法 可以 不可以
是否可以有字段 可以 只能有 public static final 常量
方法类型 抽象方法、普通方法 抽象方法、默认方法、静态方法
继承/实现关系 子类使用 extends 实现类使用 implements

3. final、finally 和 finalize 的区别？

  • final：修饰类、方法、变量，表示不能被修改（如 final class、final method、final variable）。
  • finally：异常处理的一部分，一定会执行，用于释放资源。
  • finalize()：对象被 GC 回收前会调用的方法，一般不推荐使用（已被弃用）。

4. 重载（Overload）与重写（Override）的区别？

特性 重载（Overload） 重写（Override）
是否父子类 否，可以同类中实现 是，子类重写父类方法
方法名 相同 相同
参数列表 必须不同 必须相同
返回值 可以不同 必须相同或协变
访问修饰符 无限制 子类访问修饰符 ≥ 父类的

5. Object 类常用方法有哪些？分别有什么作用？

  • toString()：返回对象字符串表示。
  • equals()：判断两个对象是否相等（地址或逻辑相等）。
  • hashCode()：返回对象哈希值，配合 equals() 使用。
  • clone()：对象浅拷贝。
  • getClass()：返回对象运行时类信息。
  • finalize()：对象被回收前调用（已过时）。

7. Java 中异常分为哪两类？它们的区别是什么？

  • 编译时异常（Checked Exception）：

    • 继承自 Exception（但不包括 RuntimeException）。
    • 必须处理（try-catch 或 throws）。
    • 如：IOException, SQLException。

  • 运行时异常（Unchecked Exception）：

    • 继承自 RuntimeException。
    • 编译器不强制处理。
    • 如：NullPointerException, ArrayIndexOutOfBoundsException。

8. throws 和 throw 的区别？

  • throw：用于抛出异常对象，只能抛一个。

  • throws：用于声明方法可能抛出的异常类型。

10. try-catch-finally 的执行顺序？在 finally 中 return 会怎样？

  • 执行顺序：try → catch（如有异常）→ finally。
  • finally 总会执行，即使 try 或 catch 中有 return。
  • 如果 finally 中也有 return，会覆盖前面的 return 值。
