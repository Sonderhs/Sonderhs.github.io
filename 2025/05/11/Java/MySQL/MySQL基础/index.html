MySQL基础

一、MySQL概述

1.1 数据库相关概念

  • 数据库：存储数据的仓库，数据是有组织的进行存储，简称DateBase(DB)
  • 数据库管理系统：操纵和管理数据库的大型软件，简称Datebase Management System(DBMS)
  • SQL：操作关系型数据库的编程语言，定义了一套操作关系型数据库统一标准，简称Structured Query Language(SQL)

1.2 MySQL安装

  • 官网：https://dev.mysql.com/downloads/mysql/
  • 启动与停止
    • 启动：net start mysql
    • 停止：net stop mysql
  • 客户端连接
    • 命令行：mysql -u root -p
    • 图形化工具：MySQL Workbench、Navicat等

1.3 数据模型

  • 关系型数据库（RDBMS）
    • 概念：建立在关系模型基础上，由多张相互连接的二维表组成的数据库
    • 特点：
      • 使用表存储数据，格式统一，便于维护
      • 使用SQL语言操作，标准统一，使用方便
  • 数据模型

二、SQL

2.1 SQL通用语法

  1. SQL语句可以单行或多行书写,以分号结尾
  2. SQL语句可以使用空格/缩进来增强语句的可读性
  3. MySQL数据库的SQL语句不区分大小写, 关键字建议使用大写
  4. 注释：
    • 单行注释: – 注释内容或#注释内容(MySQL特有)
    • 多行注释: /注释内容/

2.2 SQL语句分类

分类 全称 说明
DDL Data Definition Language 数据定义语言，用来定义数据库对象（数据库，表，字段）
DML Data Manipulation Language 数据操作语言，用来对数据库表中的数据进行增删改
DQL Data Query Language 数据查询语言，用来查询数据库中表的记录
DCL Data Control Language 数据控制语言，用来创建数据库用户、控制数据库的访问权限

2.4 DML语句

  • DML介绍
    • DML（Data Manipulation Language）数据操作语言，用来对数据库表中的数据进行增删改
    • 添加数据(INSERT)
    • 修改数据(UPDATE)
    • 删除数据(DELETE)

2.4.1 添加数据

  • DML-添加数据
    • 给指定字段添加数据
    • 给全部字段添加数据
    • 批量添加数据
    • 注意：
      • 插入数据时，指定的字段顺序需要与值的顺序是一一对应的
      • 字符串和日期型数据应该包含在引号中
      • 插入的数据大小应该在字段的规定范围内

2.4.2 修改数据

  • DML-修改数据
    • 修改指定字段
    • 修改全部字段
    • 注意：
      • 如果不加条件，所有记录都会被修改
      • 如果加了条件，只有符合条件的记录会被修改

2.4.3 删除数据

  • DML-删除数据
    • 删除指定记录
    • 删除全部记录
    • 注意：
      • 如果不加条件，所有记录都会被删除
      • 如果加了条件，只有符合条件的记录会被删除
      • DELETE语句不能删除某一个字段的值（可以使用UODATE）

2.5.4 分组查询

  • DQL-分组查询
    • GROUP BY
    • where和having的区别
      • 执行时机不同：
        • where在分组之前执行，不满足where条件的不参与分组
        • having在分组之后执行
      • 判断条件不同：
        • where不能对聚合函数进行判断
        • having可以判断聚合函数
    • 分组之后，查询的字段一般为聚合函数和分组字段，查询其他字段无任何意义

  • DQL-排序查询
    • ORDER BY

      • ASC：升序（默认）
      • DESC：降序
      • 多个字段排序时，当第一个字段相同，则根据第二个字段排序

2.5.6 分页查询

  • DQL-分页查询
    • LIMIT
    • 注意：
      • 起始索引从0开始，起始索引 = （查询页码 - 1）* 每页显示记录数
      • 分页查询是数据库的方言，不同的数据库有不同的实现，MySQL使用LIMIT
      • 如果查询的是第一页数据，起始索引可以省略，直接简写为limit 10
