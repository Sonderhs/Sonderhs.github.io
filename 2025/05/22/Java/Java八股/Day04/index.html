Day04 八股文详细答案（JVM + 内存模型）

1. JVM 的主要组成部分有哪些？

  • 类加载器（ClassLoader）：负责加载 class 文件。
  • 运行时数据区：内存结构，包括方法区、堆、虚拟机栈等。
  • 执行引擎：执行字节码（包括解释器、JIT编译器）。
  • 本地接口（Native Interface）：调用本地方法（如 C/C++ 实现）。
  • 垃圾收集器（GC）：自动管理内存。

2. Java 内存模型（JMM）是什么？

Java 内存模型定义了 线程与主内存之间的可见性、原子性、有序性规则。JMM 不是真实存在的内存结构，而是 Java 虚拟机抽象的一种模型。

例如：

  • volatile 保证变量对所有线程可见。
  • synchronized 保证互斥和可见性。
  • 指令重排由 JMM 控制。

3. JVM 内存结构主要包括哪些区域？

  1. 程序计数器：记录当前线程所执行字节码的行号指示器。
  2. 虚拟机栈：线程私有，保存栈帧，支持方法调用。
  3. 本地方法栈：用于本地方法的执行（如 C 语言）。
  4. 堆（Heap）：存放对象实例，GC管理的主要区域。
  5. 方法区（Metaspace）：存放类元数据、常量、静态变量等。

4. 什么是堆、栈、方法区，它们的作用是什么？

  • 堆：存储对象实例，是垃圾收集的重点区域。
  • 栈：存储每个线程的方法调用栈帧，局部变量等。
  • 方法区：JDK8 之后由元空间（Metaspace）取代，用于存储类信息、常量、静态变量。

5. 什么是双亲委派模型？有什么优点？

  • 定义：类加载器加载类时，优先让父加载器尝试加载，只有父加载失败，才由子加载器加载。

  • 优点：

    • 避免重复加载。
    • 保证核心类不会被篡改，如 java.lang.String。

6. 类加载过程分为哪几个阶段？

  1. 加载（Load）：通过类名找到对应字节码文件。
  2. 验证（Verify）：校验字节码文件合法性。
  3. 准备（Prepare）：为类的静态变量分配内存并初始化为默认值。
  4. 解析（Resolve）：将常量池中的符号引用转为直接引用。
  5. 初始化（Initialize）：执行 <clinit> 方法，正式赋值。

7. 强引用、软引用、弱引用、虚引用的区别？

引用类型 是否参与回收 特点
强引用 否 常规对象引用，不会被 GC 回收
软引用 是 内存不足时回收，可用于缓存
弱引用 是 GC 时一定回收
虚引用 是 不可访问，仅用于监控对象是否被回收

8. GC 的触发条件是什么？有哪些垃圾收集器？

  • 触发条件：

    • Eden 区满 → Minor GC。
    • Old 区满或系统内存不足 → Full GC。

  • 常见垃圾收集器：

    • Serial：单线程，适合小内存。
    • ParNew：Serial 多线程版。
    • CMS：并发低停顿，已淘汰。
    • G1：面向服务器，低延迟。
    • ZGC / Shenandoah：大内存、低停顿、可扩展。

9. Minor GC 和 Full GC 的区别？

GC 类型 回收区域 停顿时间 频率
Minor GC 新生代 短 频繁
Full GC 整个堆区 长 较少

Minor GC 只回收新生代，Full GC 会回收整个堆及方法区，代价大。

10. 如何排查内存泄漏？有哪些常用工具？

排查方法：

  • 使用内存快照（heap dump）对比前后差异。
  • 查找长生命周期对象中是否持有无用引用。
  • 分析线程堆栈或内存增长图。

常用工具：

  • jvisualvm：JDK 自带，直观查看内存、线程。
  • MAT（Memory Analyzer Tool）：分析泄漏、找出 GC Root。
  • YourKit / JProfiler：商业分析工具，功能强大。
